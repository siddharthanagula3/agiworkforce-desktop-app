- **pnpm:** 9.15.0+ (install: `npm install -g pnpm@9.15.3`)

### **Installation Steps**

#### **Step 1: Clone the Repository**

```bash
git clone https://github.com/siddharthanagula3/agiworkforce-desktop-app.git
cd agiworkforce-desktop-app
```

#### **Step 2: Install Dependencies**

```bash
# Install all dependencies (this may take a few minutes)
pnpm install
```

This installs:

- Frontend dependencies (React, TypeScript, etc.)
- Rust dependencies (Tauri, automation libraries, etc.)
- Shared packages (types, UI components, utilities)

#### **Step 3: Run in Development Mode**

```bash
# Start the development server
pnpm --filter @agiworkforce/desktop dev
```

**What happens:**

- Vite dev server starts (frontend)
- Tauri builds Rust backend
- Application window opens automatically
- Hot reload enabled (changes reflect immediately)

The app will open at `http://localhost:5173` with the desktop window.

#### **Step 4: Configure LLM Provider (Optional but Recommended)**

1. Open Settings in the app
2. Add your API keys:
   - **OpenAI:** Get key from https://platform.openai.com/api-keys
   - **Anthropic:** Get key from https://console.anthropic.com/
   - **Google:** Get key from https://makersuite.google.com/app/apikey
   - **Ollama:** Install from https://ollama.com (no key needed!)

**Note:** You can use Ollama (local LLM) for free - no API keys required! Data never leaves your machine.

### **Production Build**

```bash
# Build optimized production executable
pnpm --filter @agiworkforce/desktop build

# Executable will be at:
# apps/desktop/src-tauri/target/release/agiworkforce-desktop.exe
```

The production build:

- Includes all dependencies
- Optimized for performance
- Smaller file size (~15MB vs ~200MB for Electron apps)
- Ready to distribute

### **Tech Stack Explained**

#### **Frontend Layer (What You See)**

- **React 18 + TypeScript 5.4+**
  - Modern, type-safe UI framework
  - Fast rendering with concurrent features
- **Zustand**
  - Lightweight state management
  - Stores chat history, settings, tool results
- **Radix UI + Tailwind CSS**
  - Accessible, beautiful components
  - Consistent design system
- **Monaco Editor**
  - VS Code's editor (same one you're familiar with!)
  - Syntax highlighting, IntelliSense, multi-cursor
- **xterm.js**
  - Terminal emulator in the browser
  - Shows command output in real-time

#### **Backend Layer (The Engine)**

- **Tauri 2.0**
  - Desktop framework (like Electron, but 10x faster)
  - Secure IPC between frontend and backend
  - Native OS integration
- **Tokio**
  - Async runtime for Rust
  - Handles thousands of concurrent operations
- **rusqlite**
  - SQLite database for local storage
  - Stores conversations, knowledge, settings
- **reqwest**
  - HTTP client for API calls
  - Handles LLM API requests, external APIs
- **windows-rs**
  - Windows API bindings
  - Enables UI automation, system control
- **playwright**
  - Browser automation engine
  - Controls Chrome/Edge via CDP

#### **LLM Integration Layer**

- **Multi-Provider Router**
  - Routes requests to best provider
  - Handles failures, fallbacks
- **Real SSE Streaming**
  - Streams responses in real-time (like ChatGPT)
  - Shows tokens as they're generated
- **Function Calling**
  - LLM can call tools directly
  - Seamless integration with automation
- **Response Caching**
  - Caches common responses
  - Reduces API costs
- **Cost Tracking**
  - Tracks token usage per provider
  - Shows costs in real-time

#### **Option 2: Anthropic (Recommended for Complex Reasoning)**

1. Get API key from https://console.anthropic.com/
2. In AGI Workforce Settings â†’ LLM Providers â†’ Anthropic
3. Paste your API key
4. Select default model (Claude 3.5 Sonnet recommended)

**Cost:** ~$0.01-0.15 per task

#### **Option 3: Google (Recommended for Multimodal Tasks)**

1. Get API key from https://makersuite.google.com/app/apikey
2. In AGI Workforce Settings â†’ LLM Providers â†’ Google
3. Paste your API key
4. Select default model (Gemini 1.5 Pro recommended)

**Cost:** ~$0.005-0.05 per task

#### **Option 4: Ollama (FREE - Recommended for Privacy)**

1. Install Ollama: https://ollama.com/download
2. Pull a model:
   ```bash
   ollama pull llama3        # General purpose
   ollama pull codellama     # Code-focused
   ollama pull mistral       # Fast and efficient
   ```
3. Start Ollama service:
   ```bash
   ollama serve
   ```
4. In AGI Workforce Settings â†’ LLM Providers â†’ Ollama
5. Select your model

**Cost:** $0 (runs locally, data never leaves your machine!)

**Recommendation:** Use Ollama for most tasks (free), use cloud providers for complex tasks.

### **Database Setup**

- **SQLite database** is auto-created at: `%APPDATA%/agiworkforce/agiworkforce.db`
- **Migrations** run automatically on app startup
- **No configuration needed** - just works out of the box!

### **MCP Server Configuration**

AGI Workforce can connect to **any MCP (Model Context Protocol) server**:

1. Go to Settings â†’ MCP Servers
2. Add server configuration:
   ```json
   {
     "mcpServers": {
       "filesystem": {
         "command": "npx",
         "args": ["-y", "@modelcontextprotocol/server-filesystem", "."],
         "enabled": true
       }
     }
   }
   ```
3. Enable the server
4. Tools from that server become available automatically

**Popular MCP Servers:**

- `@modelcontextprotocol/server-filesystem` - File operations
- `@modelcontextprotocol/server-github` - GitHub integration
- `@modelcontextprotocol/server-gdrive` - Google Drive
- `@modelcontextprotocol/server-slack` - Slack integration
- `@modelcontextprotocol/server-brave-search` - Web search

### **Auto-Approval Settings**

For fully autonomous operation:

1. Go to Settings â†’ Agent â†’ Auto-Approve
2. Enable auto-approval for safe operations
3. Configure safety rules:
   - âœ… Auto-approve: File reads, code analysis, database queries
   - âš ï¸ Require approval: File writes, system commands, API calls
   - âŒ Never auto-approve: Delete operations, system changes

### **Context Compaction Settings**

To customize conversation management:

1. Go to Settings â†’ Chat â†’ Context Management
2. Adjust thresholds:
   - Max tokens before compaction (default: 100,000)
   - Target tokens after compaction (default: 50,000)
   - Messages to keep intact (default: 10)
