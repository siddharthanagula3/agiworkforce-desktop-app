### Multi-Agent Orchestration

#### Parallel Agent Patterns

**Parallel Agent Teams:**
- Multiple specialized agents working simultaneously
- Clear ownership boundaries
- Reduced context pollution
- Faster completion through parallelization

**Example (Cursor):**
```
User: "Add authentication to my app"
Agent 1: Supabase Auth (25s)
Agent 2: NextAuth.js (28s)
Agent 3: Auth0 (22s)
â†’ Developer reviews all 8 approaches, picks best
```

**Hierarchical Systems:**
- Master agent: High-level planning
- Sub-agents: Specialized execution
- Tool agents: Specific functions
- Flow: Delegation â†’ Execution â†’ Synthesis

#### Orchestration Strategies

**1. Sequential Pipeline:**
- Task â†’ Agent 1 â†’ Agent 2 â†’ Agent 3 â†’ Result
- Clear handoffs, simplest to debug, slowest

**2. Parallel with Join:**
- Independent agents run concurrently
- Results combined at join point
- Faster execution, requires merge logic

**3. Conditional Branching:**
- Route to different agents based on criteria
- Specialized handling per scenario
- Dynamic workflow adaptation

**4. Supervisor-Worker:**
- Supervisor breaks down complex task
- Workers execute independently
- Supervisor validates and integrates

#### State Management

**Shared Context:**
- Central knowledge store
- Prevents redundant work
- Maintains consistency
- Challenge: Concurrency control

**Isolated Context:**
- Separate context per agent
- Prevents information overload
- Clear boundaries
- Challenge: Communication overhead

**Hybrid Approach (Best Practice):**
- Shared read-only context
- Isolated write context
- Explicit handoffs for transfer
